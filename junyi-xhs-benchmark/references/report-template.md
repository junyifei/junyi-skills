# Deliverable template

## 1. Decision first

- Present five large, five medium, and five small accounts with profile URLs when evidence permits.
- Assign each a scale tier and a learning role: direct, recent breakout, one-stage-ahead, trust/story, topic, hook, packaging, or conversion.
- Separate core accounts, specialist accounts, and post-level specimens.
- State whether the list is verified, partially verified, or provisional.

## 2. Benchmark brief

Summarize the user's recall trigger, reach audience, paying buyer, content branches, commercial anchor, stage, and constraints. Label assumptions.

## 3. Search scope

List observation date, platform access condition, result slots scanned, unique authors discovered, number verified, number scored, saturation status, and sampling limits.

Report content-branch coverage:

| Trunk/branch lane | Queries run | Unique authors | New specialist/post specimens | Coverage gap |
|---|---|---:|---:|---|

Report six-path coverage:

| Path | Queries or seeds used | Pages/posts/accounts scanned | Candidates or themes found | Limits |
|---|---|---:|---|---|
| Topic pages |  |  |  |  |
| Target audience |  |  |  |  |
| Real problems |  |  |  |  |
| Content mechanisms |  |  |  |  |
| Seed expansion |  |  |  |  |
| Profile/comments |  |  |  |  |

For every core keyword, state the topic page inspected, whether popular and latest views were both checked, the number of posts scanned, and repeated authors found. State explicitly when the 30–50-post target was not met.

State whether the stopping rule was met: every branch covered and two consecutive distinct discovery runs produced no new plausibly useful authors. If not, label the discovery `not saturated`.

## 4. Seed-account fingerprint

When a user supplied a seed account, show its audience, identity, problem, desired result, topic families, formats, proof, creator temperament, and monetization path. Explain how the fingerprint generated adjacent searches.

## 7. Optional execution shortlist

For each account include:

- profile URL and dated snapshot;
- why it matches;
- recurring content and visible performance pattern;
- audience language or demand signal;
- visible trust and commercial path;
- what to learn;
- what not to copy;
- unknowns and confidence.

## 8. Cross-account map

|---|---|---|---|

## 9. Tests for the user's account

Turn observations into at least three content hypotheses. For each specify stimulus, intended audience, primary metric, comparison baseline, sample size or duration, and pass/mixed/fail next action.

## 10. Evidence boundary

State what public data cannot establish: customer quality, revenue, payment, retention, private-message volume, or causal algorithm effects.
