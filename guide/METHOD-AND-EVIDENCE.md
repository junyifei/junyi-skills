# 方法与证据说明

君一方法论不以 Skill 数量作为成熟度证明。一个目录能运行，不等于方法已经有效；一条内容受到关注，也不等于产品和交付已经成立。

## 四类陈述

| 标签 | 含义 | 可以支持什么 |
|---|---|---|
| 事实 | 有原始记录、可观察结果或可靠来源 | 说明已经发生的有限事实 |
| 推断 | 由事实支持的当前解释 | 帮助做暂定判断，不能写成已验证 |
| 假设 | 可以用行动检验、尚未被证明 | 设计下一次实验 |
| 未知 | 做决定需要、当前没有的信息 | 明确停止夸大的位置 |

## 方法来源

公开方法主要来自：

- 一人公司经营中的真实任务与决定；
- 内容、知识、课程和运营中的 Agent 工作流；
- 课程、咨询、访谈和用户问答；
- 家庭实践和长期成长记录；
- 好结果、差结果以及君一亲自修改过的版本。

这些来源能说明方法不是凭空编写，不能自动证明每个公开 Skill 都能带来流量、收入、关系或成长结果。

## 晋升为正式 Skill 的质量门

一个候选方法至少需要：

1. 清楚的触发任务和不适用边界；
2. 可以重复执行的步骤，而不只是观点；
3. 明确的输入、证据和交付物；
4. 真实任务记录，包含失败或人工修改；
5. 隐私、权利、路径和外部依赖检查；
6. 完成、失败与补充信息的验收标准；
7. 能够独立安装，且目录名与 frontmatter 一致。

没有通过质量门的方法保留在本地知识库或待验证区，不进入公开索引。

## 成熟度含义

| 成熟度 | 含义 | 不代表什么 |
|---|---|---|
| 已发布 | 结构、边界和基本任务已经验证 | 不代表适合所有人或所有场景 |
| 已投入真实项目 | 已用于真实项目并形成可检查产物 | 不代表商业结果可以复制 |
| 测试中 | 方法和交付结构已经建立，仍需更多真实任务校准 | 不代表已经稳定有效 |

## 当前公开 IP 能力能说到什么程度

### `junyi-positioning`

可以说：已投入真实个人 IP 项目，用于区分证据、定位层、目标人群、能力证明和验证计划。

不能说：已经证明可以帮助任何人涨粉、成交或建立百万粉丝账号。

### `junyi-xhs-benchmark`

可以说：已建立宽召回、核验、分层、评分和排除逻辑，并进入测试。

不能说：推荐的账号一定可复制，或使用报告一定能产生爆款。

## 反馈怎样进入版本

优先记录以下材料：

- 用户原始任务与输入类型；
- 输出中真正被采用的部分；
- 错误、遗漏和不适用场景；
- 人工修改前后差异；
- 修改后是否在真实任务中更可用；
- 哪些材料允许公开，哪些只能保留为私有证据。

关注、Stars 和下载量只能说明采用或注意力，不替代任务结果与人工修改记录。
