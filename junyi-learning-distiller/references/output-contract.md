## 固定骨架

```markdown
# 学习蒸馏｜材料名称

> 来源：作者/主讲人，材料标题，日期/版本
> 覆盖范围：完整 / 指定章节 / 指定时间段
> 个人应用背景：已提供 / 未提供
> 外部核验：未进行 / 已进行（另列来源）

## L1｜关键结论

## L2｜主张与证据地图

## L3｜用自己的话讲明白

## L4｜可迁移方法

## L5｜与我已有认知的关系

## L6｜应用与小实验

## L7｜存疑与待跟进

## L8｜一句话本质
```

八层必须全部出现并保持顺序。某层没有内容时说明缺少什么，不要删除栏目或编造内容。

## 证据条目

```markdown
### C01｜主张标题
**类型**：【来源主张】
**证据位置**：第 12 页 / 00:14:20–00:16:05 / 第 4 章第 3 段
**来源依据**：简要转述；必要时附一小段原话
**支持状态**：来源内有支持 / 支持不足 / 已外部核验 / 待核验
**限制**：样本、前提、反例或未知
```

关键条目都要有证据位置。“我觉得有道理”不是证据。

## 方法条目

```markdown
### M01｜方法名称
**目标**：
**前置条件**：
**步骤**：1. … 2. … 3. …
**适用场景**：
**不适用场景**：
**常见失败方式**：
**证据位置**：
```

来源没有完整给出步骤时标 `【我的方法假设】`。

## 实验条目

```markdown
### E01｜实验名称
**要验证的假设**：
**真实场景**：
**最小动作**：
**观察信号**：
**停止/调整条件**：
**复盘时间**：
```

使用者没有提供应用背景时，写“待使用者选择真实场景”，不要虚构业务和日期。

## 长材料中间产物

每个 chunk 只生成证据账本，不生成八份局部总结：

```markdown
# Chunk 001 证据账本
> 来源范围：第 1–12 页 / 00:00–00:24

## 关键主张
### C001｜标题
**类型**：【来源主张】
**证据位置**：
**来源依据**：
**限制/待核验**：

## 方法候选
## 术语与定义
## 案例与数据
## 存疑与待回看
```

最终八层学习转化来自所有 chunk 证据账本的综合，不把各 chunk 摘要直接拼接。
