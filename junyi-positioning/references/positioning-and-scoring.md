## 1. 五层定位为什么必须分开

一句话承载过多任务时，常见结果是标签堆叠、抽象口号或买方错位。把五个任务分开，再检查它们是否共同指向同一变化。

| 层 | 回答的问题 | 合格标准 | 常见错误 |
|---|---|---|---|
| 传播入口 | 陌生人为什么停下来 | 宽、真、可见、反复发生 | 只写专业品类或自我身份 |
| 人物标签 | 别人怎样最短记住并转述 | 自然、具体、不过载 | 把所有履历塞进去 |
| 内容承诺 | 为什么值得持续关注 | 有稳定栏目、进展或可重复价值 | 只承诺“分享成长” |
| 商业落点 | 谁为何付钱买什么结果 | 买方、问题、结果、产品和边界清楚 | 把所有粉丝当买方 |
| 长期价值观 | 为什么长期信任这个人 | 能解释选择、边界和独特判断 | 抽象词替代真实选择 |

五层的连接检查：

1. 传播入口吸引的人里，是否能筛出真正买方？
2. 人物标签是否能帮助记忆，而不是限制长期发展？
3. 内容承诺是否持续产生与用户变化有关的内容？
4. 商业落点是否由能力证明承接，而不是靠人设跨越？
5. 长期价值观是否从真实决定中出现，而不是写在口号里？

## 2. 三类人群

### 触达人群

描述公开内容最容易被谁看懂。写具体处境、共同张力和平台行为，不只写年龄、城市或性别。

### 真正买方

至少写清：

- 正在发生的高成本问题；
- 已经尝试过什么；
- 预算或资源；
- 决策权与参与意愿；
- 期望的可观察结果；
- 适配条件和拒绝条件。

当购买者、使用者和受益者不同，分别记录。

### 不需要吸引的人

排除不是嫌弃用户，而是保护定位、交付和双方预期。排除依据应是需求、行为、风险或适配性，不是贬低某种身份。

检查三类人之间的漏斗：

```text
传播入口 → 愿意持续关注 → 出现具体问题 → 相信能力 → 符合买方条件 → 支付 → 获得结果
```

如果公开内容持续吸引无法进入后续路径的人，调整入口或语言；不要用更激进的销售硬拉错误人群。

## 3. 用户变化地图

按下表写“之前—关键机制—之后—证据”。结果必须可观察；心理变化可保留，但不能成为唯一结果。

| 维度 | 之前 | 关键机制/行动 | 之后 | 如何观察 | 谁能负责 |
|---|---|---|---|---|---|
| 处境与行为 | 现在怎么做 | 改变什么动作 | 新行为 | 频率/完成记录 | 用户/交付方 |
| 结果与成本 | 时间、金钱、机会成本 | 哪一步减少损耗 | 可量化结果 | 前后数据 | 共同 |
| 能力与系统 | 缺什么判断或方法 | 练习/流程/工具 | 能独立做什么 | 作品/演示/复测 | 用户 |
| 感受与身份 | 害怕、混乱、失控 | 什么经历改变解释 | 更清楚/更稳定 | 访谈/自评 | 用户 |
| 关系与边界 | 谁在承担、什么被牺牲 | 重新分工或停止 | 保护什么 | 行为证据 | 多方 |

检查：

- “之后”是否由定位或产品真的能影响？
- 是否把收入、健康、婚姻等多因素结果承诺过满？
- 是否能从变化地图直接产生内容、能力证明和交付步骤？
- 是否写了“保持不变”的尊严、隐私、责任和价值边界？

## 4. 候选方向卡

为每个候选填写同一张卡，避免用不同信息量比较：

| 字段 | 内容 |
|---|---|
| 候选名 | 自然语言，不追求口号 |
| 用户在什么情况下会想起你 | 具体时刻、后果和期望变化 |
| 触达人群 | 谁容易理解和传播 |
| 真正买方 | 谁有成本、预算、权力与意愿 |
| 不需要吸引的人 | 错误需求与边界 |
| 用户变化 | 之前 → 之后 |
| 传播入口 | 宽问题、冲突或好奇 |
| 人物标签 | 最短记忆 |
| 内容承诺 | 持续价值/连续剧情 |
| 商业落点 | 买方、问题、结果、产品 |
| 长期价值观 | 判断与边界 |
| 独有资产 | 经历、能力、场景、证据 |
| 能力证明 | 结果、过程、案例、产物 |
| 100 题燃料 | 五类题库与数量 |
| 主要风险 | 最可能失败在哪里 |
| 证据状态 | 事实/推断/假设/未知 |
| 什么情况说明不成立 | 预先登记的条件 |

候选之间至少在用户情况、变化、买方或商业落点之一有实质差异。只替换几个身份词不算不同方向。

## 5. 成为主定位前必须通过的基础检查

先检查，再评分。允许 `通过 / 暂定 / 不通过 / 未知`；其中“暂定”必须带实验，“不通过”不能靠高总分晋级。

### 检查 1：用户想起你的情况

通过：能写出一个具体、重复、对用户有后果的时刻。

不通过：只有“女性成长”“想变好”“创业者”等人群或抽象愿望。

### 检查 2：100 题真实燃料

从五类材料测试：

- 真实现场与连续进展；
- 方法、演示和产物；
- 错误、冲突和决定；
- 用户问题、案例和比较；
- 观点、文化变化和长期探索。

至少四类各有 15 题，总计不少于 100。大部分题目若只能反复消费旧创伤，则不通过。

### 检查 3：一句话转述

用自然句式测试：

> 如果你正在___，可以看看___；他/她会持续展示或帮助你___。

如果必须解释三个以上标签或使用用户听不懂的术语，则不通过。

### 检查 4：能力可信

至少有两种证据，或能在短期内真实产生：本人结果、客户结果、过程演示、可复用产物、证书加实际应用、重复交付、留存或转介绍。

被理解和喜欢不能单独通过这项检查。

### 检查 5：支付邻近

已为同一结果付款、付定金、复购或转介绍才可标“已验证”。没有付款时可暂定，但必须设计最小付费实验。旧产品付款只能支持旧产品对应的结果。

### 检查 6：伦理与生活匹配

方向不应依赖虚假恐惧、隐私暴露、羞辱用户、无法承受的表演、本人不想经营的业务或不可持续产能。

## 6. 加权评分

基础检查之后，每项按 0-5 分评分。分数必须附证据编号或“未知”，不要用小数制造精确幻觉。

| 维度 | 权重 | 问题 |
|---|---:|---|
| 用户情况强度 | 15 | 是否频繁、具体、可见、有后果？ |
| 个人资产拥有度 | 10 | 是否真有经历、能力或场景？ |
| 内容燃料 | 10 | 是否能持续 100 题并随实践更新？ |
| 转述清晰度 | 10 | 陌生人能否准确复述？ |
| 注意力证据 | 10 | 相关内容是否吸引了合适的人？ |
| 能力证明 | 15 | 是否相信能交付变化？ |
| 支付邻近 | 15 | 问题是否值得付钱且有合适产品？ |
| 规模上限 | 10 | 传播入口是否连接足够广的现实张力？ |
| 伦理与生活匹配 | 5 | 是否符合边界和长期生活？ |

计算：`总分 = Σ(单项分 / 5 × 权重)`。

需要脚本计算时，把候选写成以下 JSON 结构；基础检查状态只使用 `pass`、`provisional`、`fail`、`unknown`：

```json
{
  "candidates": [
    {
      "name": "候选方向",
      "gates": {
        "recall_situation": "pass",
        "content_fuel_100": "pass",
        "retell_sentence": "pass",
        "ability_trust": "provisional",
        "payment_test": "unknown",
        "ethical_fit": "pass"
      },
      "scores": {
        "trigger_strength": 4,
        "asset_ownership": 4,
        "content_fuel": 4,
        "retell_clarity": 4,
        "attention_evidence": 2,
        "ability_proof": 3,
        "payment_adjacency": 2,
        "scale_ceiling": 3,
        "ethical_fit": 5
      },
      "evidence": ["E001", "H001"]
    }
  ]
}
```

参考解释：

- 80-100：强候选；没有新主张付款时仍只能写“暂定赢家”。
- 65-79：有潜力；先修 1-2 个弱层。
- 50-64：只做受控挑战方向，不投入整套账号。
- 50 以下：放弃或回到资产与用户情况。

分数只帮助比较，不能把未知变成确定，也不能替代基础检查或付款。

## 7. 如何选赢家

最终决策至少包括：

1. 推荐主方向；
2. 最多一个挑战方向；
3. 推荐方向通过了哪些基础检查；
4. 主要证据编号；
5. 最大未知；
6. 选择它而不是高分相邻方向的关键取舍；
7. 30 天最小验证；
8. 什么情况说明这条路不成立；
9. 失败时先修改哪一层。

分数接近时，优先选择证据更强、真实素材更多、能力证明更快、生活更可持续的方向，而不是语言更宏大的一项。
