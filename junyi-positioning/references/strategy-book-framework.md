## 1. 一个事实源

完整《IP战略书》同时服务两类阅读：决策者先看当前定位，研究者和执行者继续查看证据、取舍、边界与验证。不要另外维护“快速定位”“简化版定位”或“执行档案”。

`B00｜定位基准` 是当前正式决定；`B01`—`B11` 解释决定依据、限制和修订条件。B00 不是第二份摘要文件，而是同一本战略书的结论章节。

## 4. 不同任务的增补

### 从零设计

完整输出 B00–B11。证据不足时 B00 仍可写暂定决定，但必须在 B01、B05、B10、B11 绑定未知和实验。

### 审核现有定位

在阅读导航后增加审计结论：保留什么、哪个层先失败、证据问题还是表达问题、最小修复、暂时不改什么。仍使用 B00–B11，不另建第二套审计结构。

### 迭代旧战略书

在阅读导航后增加版本差异表：旧决定、新证据、新判断和动作。保留仍成立的内容，不为制造新版而全盘重写。

### 只做诊断

可以只输出失败层、证据缺口、候选比较、最小修复和验证条件，但必须明确“这不是完整《IP战略书》”，也不能把诊断结果保存成另一个正式定位事实源。

## 5. 输出与下游读取

- 默认文件名为 `IP战略书.md`，版本写在文件内部或由使用者现有版本规则管理。
- 先完成 Markdown 内容与证据核验，再做 Word/PDF 排版。
- 下游 Skill 直接读取本书：先完整读取 B00 与 B09，再按自己的任务读取相关章节。
- 下游 Skill 的必读章节由下游自己声明；战略书不集中维护未来所有 Skill 的路由。
- 机器结构检查通过不代表事实已经核验，最终仍要回到 B01 证据账本。
