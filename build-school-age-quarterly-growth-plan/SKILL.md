# 72—144 月龄季度成长计划

把全年规划作为长期知识底座，把本季新资料转成未来 90 天的家庭行动指南。计划既要让家长能执行，也要让孩子用适龄方式参与确认。

## 1. 校验年龄与资料

必须取得：

1. `junyi-child-growth-intake/v1` 的全年 `intake.json`；
2. 完整全年规划；
3. `junyi-child-quarterly-update/v1` 的 `quarterly-update.md`，且 `status: ready`。

直接收到原始语料时，先使用 `collect-child-quarterly-update` 提取和验证。资料不足时返回 7—14 天预备观察方案，不生成低证据正式计划。

按季度开始日计算完整月龄。只有 72—144 月龄可以继续；季度中途超过 144 月龄仍完成当前季度。全年底座来自旧轨道时，只要仍有效且资料记录了过渡，可以作为历史知识使用。

```bash
python scripts/validate_bundle.py \
  --intake path/to/intake.json \
  --update path/to/quarterly-update.md \
  --annual-plan path/to/annual-plan.md
```

## 2. 提炼五维度画像

读取：

- [references/school-age-method.md](references/school-age-method.md)
- [references/decision-method.md](references/decision-method.md)
- [references/evidence-and-safety.md](references/evidence-and-safety.md)

从全年底座与本季新资料中提炼：家庭教育目标、孩子优势与挑战、重要陪伴者风格与现实投入、现有资源、本季新数据。关键事实绑定 `Exx` 或 `Qxx`。

72—107 月龄至少取得一项简单直接反馈，不要求系统复盘整份计划；108—144 月龄需要对保留、停止或尝试什么给出较完整的直接表达和理由。成人解释不能冒充孩子声音。

## 3. 选择主攻与维持方向

按 [references/decision-method.md](references/decision-method.md) 选择：

- 主攻方向 1—2 个；
- 维持方向 1—2 个。

透明比较证据强度、时间窗口、孩子动力、家庭可执行性和投入回报。每个主攻写清当前位置、孩子反馈、为什么现在、三个月后的可观察画面，以及为什么其他方向暂不加码。没有明显紧迫方向时，优先顺着孩子当前最强且持续的动力，而不是制造问题。

## 4. 生成季度行动方案

严格使用 [references/output-format.md](references/output-format.md)。基础计划通常约 5000—8000 个中文字符；确需一个专项学习方案时可以更长。

- 没有共同选择证据时，把“用适龄方式与孩子确认”列为第 0 步；
- 每周核心行动最多 4 个，具体到第一步、频率、时长、负责人、启动前提和孩子可怎样调整；
- 必须给出“至少做到”的最低有效版本；
- 已稳定执行的兴趣、阅读、运动和家庭活动写成维持方式，不伪装为新增任务；
- 只有主攻方向确实涉及学科，才生成完整学习方案；
- 学科方案包含当前水平与置信度、核验后的材料、一次怎么做、分阶段路线、课程选择标准和评估方式；
- 产品、课程、价格和服务状态必须核验，无法核验时只写类别或 `[待确认]`；
- 根据真实家庭结构设计陪伴者任务，不默认父亲角色；
- 家长话术至少覆盖 5 个高频摩擦场景；
- 写 3—5 条个性化避坑提醒。

## 5. 验收与交付

```bash
python scripts/validate_plan.py path/to/quarterly-plan.md
```

人工检查：

- 关键判断是否绑定 `Exx` 或 `Qxx`；
- 孩子原话是否真实，是否由成人代写；
- 主攻与维持是否各 1—2 个，核心行动是否不超过 4 个；
- 是否把单次表现写成能力、人格、天赋或固定学习类型；
- 是否把作业完成、分数或服从当作唯一成功指标；
- 学科专项是否真的由学科主攻触发；
- 具体产品、课程、价格与服务是否已核验或标为待确认；
- 家庭成员与资源是否有证据，是否替他人承诺；
- 安全或专业问题是否被错误包装成家庭行动实验。

未经用户明确要求，不写入飞书、Obsidian、学校系统、客户文件夹或其他外部系统。
