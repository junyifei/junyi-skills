# 小红书数据校准

当前阶段：冷启动。外部案例决定测试方向，当前账号自己的可比较数据决定后续优先级。

## 每条记录字段

- 发布日期与内容类型；
- 内容方向与传播任务；
- 封面逐行文案和笔记标题；
- 主题、角度、主要证据；
- 曝光、播放或阅读；
- 封面点击或进入率（后台有则记录）；
- 点赞、收藏、评论、分享；
- 主页访问、关注、搜索进入（后台有则记录）；
- 合格评论或私信：是否来自目标创业者父母，具体问了什么；
- 制作成本、发布时间和内容质量异常。

## 复盘规则

1. 只与相近主题、形式、时长和账号阶段的内容比较。
2. 不把标题效果与选题、内容、封面画面、发布时间强行拆成单一因果。
3. 同一种标题结构至少测试 3 条，再决定保留、改写或停止。
4. 优先看收藏、搜索进入、主页访问、关注和合格问题是否完成本条任务，不只看点赞。
5. 单条爆款只产生新假设，不直接升级为固定公式。

## 更新 Skill 的门槛

- `12–19 条`：只调整角度排序和禁用词，不删除整个方法分支。
- `20–39 条`：建立当前账号自己的高/中/低表现样本，更新案例库。
- `40 条以上`：按内容方向和传播任务分别计算基线，重设评分权重。

更新时保留原始数据和观察日期，明确区分事实、推断与假设。
