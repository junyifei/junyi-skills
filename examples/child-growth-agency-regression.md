# 儿童全年规划回归案例：不能把 Agent 的方案写成孩子共同设计

> 本页使用合成情境说明一次严重缺陷及修复，不包含真实儿童、家庭、学校或内部报告。它证明规则与回归结果，不证明任何儿童发展结果。

## 输入情境

一份学龄儿童资料包包含孩子对兴趣和困难的表达，也包含家长观察，但没有证据显示孩子看过、选择或共同制定了 Agent 即将生成的 90 天行动实验。

## 错误输出

早期生成器可能因为资料中存在“孩子原话”，就把报告写成：

```text
孩子共同选定了本季度目标，并参与设计了行动方案。
```

这句话超出了证据。表达过兴趣，不等于看过方案；提供原话，也不等于共同设计。

## 人工修改

- 只有原始材料明确显示孩子实际参与选择，才能写“共同设计、共同选定、孩子选择”；
- 成人代述或孩子只表达笼统愿望时，必须标明来源；
- 尚未确认的方案写成“待与孩子当面共同确认”的前瞻步骤；
- “共同行动实验”只说明可能由家庭、孩子和学校协作执行，不证明孩子已经共同设计；
- 单次观察只能形成待复现提示，不能升级为稳定特质。

## 可检查结果

该缺陷修复后，全年四个 Skill 完成 15 个匿名合成案例前向回归，并对 3 个关键学龄案例进行独立重跑；公开 Skill 与校验器都包含孩子能动性、来源和确定性规则。

## 边界

- 回归通过不代表规划适合每个家庭；
- 规划不替代医疗、心理、发育筛查或教育专业评估；
- 合成案例不能作为真实家庭成效、孩子参与度或长期结果的证据。
