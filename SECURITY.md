# 安全与隐私

## 不要提交到公开仓库或 Issue

- API Key、Cookie、Token、密码和账号凭据；
- 客户、学员、孩子或家庭成员的身份信息；
- 私有飞书、网盘、知识库和本地绝对路径；
- 未获授权的聊天记录、咨询记录、课程材料和案例；
- 能够反向识别个人的组合信息；
- 尚未公开的商业数据、合同、价格和内部流程。

提交问题前，请用合成示例或最小脱敏片段复现。删除姓名不一定等于完成脱敏；时间、地点、职业、家庭结构和具体事件组合起来仍可能识别个人。

## 凭据问题

如果发现真实凭据已经进入 Git 历史，不要只修改当前文件。立即吊销或轮换凭据，再通过仓库所有者可见的安全渠道报告。公开 Issue 中不要粘贴凭据本身。

## Skill 权限

Skill 会由用户选择的 Agent 执行，并可能继承该 Agent 的文件、网络和外部工具权限。安装前应阅读 `SKILL.md`、scripts 和依赖说明；只在理解权限边界后使用。

## 结果边界

本仓库不承诺收入、粉丝、健康、关系、儿童发展或平台传播结果。涉及医疗、法律、财务、儿童安全和其他高风险判断时，应使用合格专业支持，不把 Agent 输出当作诊断或最终决定。
