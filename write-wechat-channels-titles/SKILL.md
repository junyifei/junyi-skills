# 视频号封面与标题

把真实事件包装成有关系方向、可被转发讨论的视频号封面与标题。不要把标题做成孤立金句，也不要用外部案例替当前创作者编故事。

## 读取上下文

1. 每次读取 [platform-method-and-cases.md](references/platform-method-and-cases.md)。
2. 用户提供定位、语气指南、禁用词或账号历史数据时，将其作为当前账号上下文，不假设 Skill 内置了某位创作者的人设。
3. 当存在当前账号的视频号发布数据或用户要求复盘时，读取 [calibration.md](references/calibration.md)。
4. 原始事件记录、逐字稿和可核验材料优先于外部方法。

## 输入不足时

最低输入应包含一个真实事件、一个已写好的口播稿，或一份带证据的内容简报。只有抽象观点时，先寻找能承载观点的第一方事件；找不到就生成有边界的观点候选并标记 `证据不足`，不能制造戏剧。

只有缺失信息会改变事件顺序、人物授权、结果强度或标题真实性时才询问。不得补写不存在的动作、第一反应、对话、家庭反应、客户结果或时间线。

## 工作流

### 1. 建立事件与证据账本

提取并编号：

- `事实`：谁在什么情境做了什么；
- `事件`：预期、反差、选择与后果；
- `原话`：可直接使用的句子；
- `判断`：这件事支持到什么程度；
- `未知`：标题必须避开的空白；
- `隐私/授权`：哪些人或材料不能公开。

### 2. 校准主张强度

使用证据完全支持的最弱层级：

1. `可核验事实`；
2. `有边界的个人判断`；
3. `变化主张`，必须有前后对照或重复观察；
4. `普遍因果`，没有广泛证据时必须降级。

标题不能比口播和证据更激进。

### 3. 确定传播任务与关系对象

选择一个主任务：`被发现 / 建立理解 / 证明能力 / 建立信任 / 促进咨询`。

再写清：观众可能转给谁，想和对方讨论什么。视频号标题应帮助关系中的人说出一句难以直接说的话，而不是泛泛追求“转发率”。

### 4. 找到前三秒冲突

从真实材料中提炼：

- 人物；
- 责任或关系；
- 当时发生的具体冲突；
- 创作者做了或拒绝做的选择。

开头与标题必须指向同一事件。没有真实行动或证据时，用有边界的陈述开场，不制造“那天突然”“第一反应”等戏剧。

### 5. 生成不同标题方向

默认生成 4 套；证据较少时至少 3 套并说明原因。优先覆盖：

1. `关系责任冲突`：创业、伴侣、父母、孩子或团队之间的责任张力；
2. `事件选择后果`：发生了什么、创作者怎样选、留下什么结果；
3. `人的判断`：工具能做什么，但什么仍必须由人决定；
4. `身份连续剧`：创作者长期项目的真实进展、翻车、调整或停用。

每套必须写明 `relationship_target`、`discussion_question`、`claim_strength` 和 `source_ids`。

### 6. 分开封面、标题与前三秒

- 封面负责一眼看懂人物、关系或冲突；写 1–3 行，每行编辑目标不超过 10 个非空白字符。
- 标题补充事件、选择、后果或人的判断；编辑目标不超过 30 个非空白字符。
- `three_second_line` 必须能自然接住标题，不得出现标题有、视频没有的承诺。
- 封面和标题互补，不能把同一句话重复两遍。
- 默认不用 emoji；这是本 Skill 的编辑约束，不冒充平台硬规则，用户可以用自己的品牌规范覆盖。
- 不用奖励、威胁、羞耻或命令诱导分享、关注、点赞和评论。

### 7. 评分与淘汰

每套按 100 分评分：

| 维度 | 分值 |
|---|---:|
| 真实事件、证据与主张匹配 | 25 |
| 关系对象与责任冲突 | 20 |
| 事件具体性与画面 | 15 |
| 转发理由与讨论增量 | 15 |
| 创作者定位与口吻 | 10 |
| 封面、标题、前三秒一致互补 | 10 |
| 当前平台合规 | 5 |

以下情况直接淘汰：标题正文不一致；虚构冲突或结果；未经授权暴露第三方或孩子；假惨、煽动对立、贩卖焦虑；保证收入或效果；把外部热门视频改写成换词版；用普遍因果包装单一事件。

### 8. 输出并校验

先给简洁可读版，再附可校验 JSON：

```json
{
  "platform": "video",
  "source_summary": "",
  "primary_task": "建立理解",
  "intended_recipient": "",
  "evidence": [
    {"id": "E1", "type": "事件", "text": ""}
  ],
  "options": [
    {
      "id": "V1",
      "angle": "关系责任冲突",
      "cover": ["行1", "行2"],
      "title": "标题",
      "title_chars": 0,
      "three_second_line": "",
      "relationship_target": "",
      "discussion_question": "",
      "claim_strength": "有边界的个人判断",
      "source_ids": ["E1"],
      "score": 0,
      "risk": ""
    }
  ],
  "recommended_id": "V1",
  "test_hypothesis": ""
}
```

运行：

```bash
python3 scripts/validate_titles.py result.json
```

若有原文文件：

```bash
python3 scripts/validate_titles.py result.json --source source.txt
```

修正所有错误后再交付。程序无法判断完整的事件真伪、授权和标题正文一致性，这三项必须人工复核。

## 数据更新边界

当前外部案例只支持关系故事与事件表达假设，不代表视频号推荐公式。不得声称固定算法权重、最佳时长或保证爆款。当前账号积累至少 12 条可比较的视频号数据后，按 [calibration.md](references/calibration.md) 更新优先级。
