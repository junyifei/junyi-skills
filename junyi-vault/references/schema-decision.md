# Schema 强度决策

## 强 schema

适合字段稳定、以后需要比较或统计的内容，如规范化案例、实验记录、项目复盘。

条件：主要信息能切成独立字段，字段含义稳定，拆分不会破坏原话和叙事。

## 半 schema

适合咨询记录、访谈、学习转化和带原话的复盘。先保留背景、变化、行动等骨架，再保存关键原话和上下文。

## 无 schema

适合故事、观点、信件、日记、完整叙事和表达性文本。保留原貌，只加来源、日期、标题和必要索引。

## 决策问题

1. 未来会不会把多条记录放在一起比较？
2. 信息能否独立装进稳定字段？
3. 拆开后是否会损失语气、关系、时间线或因果？
4. 当前库是否已有同类模板？

现有模板优先。缺失字段写未知，不编造；有力量的原话优先于填满字段。
