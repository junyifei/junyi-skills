# 0—35 月龄季度成长计划

把全年规划作为长期知识底座，把最近 6—8 周及本季度新增资料转成家长未来 90 天能执行的观察与环境方案。家长报告帮助家长理解孩子、提供环境，不重新评估孩子是否“达标”。

## 1. 校验年龄与资料

必须取得：

1. `junyi-child-growth-intake/v1` 的全年 `intake.json`；
2. 完整全年规划；
3. `junyi-child-quarterly-update/v1` 的 `quarterly-update.md`，且 `status: ready`。

如果用户直接提供原始语料，先使用 `junyi-child-quarterly-intake` 提取和验证；资料充分时可在当前任务内完成，不要求用户另下命令。资料不足时返回 7—14 天预备观察方案，不生成低证据正式计划。

按季度开始日计算完整月龄。只有 0—35 月龄可以继续；季度中途跨过 36 月龄不换 Skill。全年底座轨道不同不自动停机，但必须确认旧底座仍有效，并在本季更新中记录年龄轨道变化。

```bash
python scripts/validate_bundle.py \
  --intake path/to/intake.json \
  --update path/to/quarterly-update.md \
  --annual-plan path/to/annual-plan.md
```

## 2. 提炼五维度画像

读取：

- [references/infant-method.md](references/infant-method.md)
- [references/decision-method.md](references/decision-method.md)
- [references/evidence-and-safety.md](references/evidence-and-safety.md)

从全年底座和本季新资料中提炼：

1. 家庭教育目标与底线；
2. 孩子的优势与成长中课题；
3. 主要及其他陪伴者的风格、优势与现实投入；
4. 已有时间、环境、社交、照护与季度事件资源；
5. 最近 6—8 周的变化、萌芽、跳跃、卡点与环境变化。

关键事实绑定 `Exx` 或 `Qxx`。优势篇幅应明显多于课题篇幅；课题写成“正在发展中的能力”，不写成缺陷标签。

## 3. 确定观察线与环境方案

本年龄段不设置主攻/维持双轨。按 [references/decision-method.md](references/decision-method.md)：

- 从萌芽信号、跳跃信号、环境信号和有可靠来源的年龄窗口中筛选 3—5 条观察线；
- 为每条观察线写明观察什么、已有证据和本季度要回答的 2—3 个观察问题；
- 生成 3—5 个环境提供方案，说明为什么、怎么做、如何支持观察线；
- 只改变成人回应、环境、节奏和可参与步骤，不给婴幼儿安排密集训练或学科目标。

单次事件值得如实记录，可以成为低风险观察的起点，但不能被写成稳定能力、人格或发展结论。

## 4. 生成家长行动指南

严格使用 [references/output-format.md](references/output-format.md)。基础计划通常约 4000—7000 个中文字符；资料丰富时可更长，但不得重复堆砌。

- 用具体场景写家庭画像和孩子优势，不用抽象标签；
- 用“提供什么环境、成人怎样回应、观察什么变化”代替训练清单；
- 把建议嵌入进食、睡眠、游戏、散步、清洁、穿脱和出门等真实生活；
- 根据真实家庭结构设计主要或其他陪伴者专属互动，不默认存在父亲角色；
- 家长话术覆盖 5—8 个高频场景，直接可开口；
- 写 3—5 条与该家庭证据相关的避坑提醒；
- 观察指标只描述本计划中的变化、停滞和求助边界，不使用同龄量尺给孩子定性；
- 每月三个复盘问题分别关注孩子体验、观察到的变化和家庭负担。

## 5. 验收与交付

```bash
python scripts/validate_plan.py path/to/quarterly-plan.md
```

机器通过后人工检查：

- 所有关键判断是否有 `Exx` 或 `Qxx` 支持；
- 是否出现了主攻/维持、训练、达标、追赶或提前学习；
- 是否把单次行为升格为能力、人格、天赋或发展结论；
- 是否把家长解释冒充孩子偏好；
- 观察线和环境方案是否各为 3—5 个；
- 家庭画像、成员分工与资源是否来自真实资料；
- 是否编造接送、陪伴、见面、课程或第三方承诺；
- 是否存在尚未处理的安全或专业问题。

未经用户明确要求，不写入飞书、Obsidian、客户文件夹或其他外部系统。
