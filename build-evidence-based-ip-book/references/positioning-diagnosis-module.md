## 1. 何时启用

当用户只需要个人 IP 定位设计、定位比较、定位审核、定位收窄或定位改版，而不需要立即生成完整战略书时，启用本模块。

本模块仍属于 `build-evidence-based-ip-book`。它复用同一份证据账本、六项基础检查和加权评分，不维护第二套定位标准。

如果定位结论需要继续展开为内容系统、产品阶梯、商业验证和 30/90 天路线，回到完整战略书流程。

## 2. 定位诊断的核心对象

把定位视为一个“被想起和被转述的约定”，而不是创作者自我介绍：

> 当谁遇到什么具体情况、想得到什么变化时，应该想起这个人？

一个可用定位至少满足：

- 能用一句自然语言转述；
- 会被具体、重复发生的情况触发；
- 有真实资产或明确的能力证明路径；
- 能持续产生来自真实实践的内容；
- 与一个值得解决、能够承接的付费问题相邻；
- 不依赖隐私消费、虚假恐惧或本人不想经营的生活方式。

身份、履历和人格是定位材料，不自动等于定位。先选择用户情况和变化，再决定哪些身份用于增加记忆、证明和故事深度。

## 3. 三种定位任务

### 设计

从个人资产、能力证明、需求信号、限制和未知中生成 3-5 个实质不同的候选方向。候选必须在用户情况、变化、买方或商业落点上存在差异，不能只重排身份标签。

### 审核

保留现有定位作为被审对象。判断问题首先出现在哪一层：用户情况、传播入口、人物标签、内容承诺、能力证明、商业落点、产品或表达。不要因为一句文案不顺就重做整套定位。

### 迭代

使用多次受控内容实验、合格咨询、付费与交付结果，只修改反复失败的层。新增证据没有推翻旧决定时，保留旧决定。

## 4. 诊断顺序

### 第一步：核验证据边界

将重要陈述标为事实、推断、假设或未知。先排除用播放证明付款、用故事证明能力、用旧产品成交证明新定位等证据越级。

### 第二步：写出用户想起你的情况

要求出现具体人物、正在发生的情况、后果和期望变化。只有人口属性、行业名词或抽象愿望时，先返回这一层修复。

### 第三步：检查五层是否连接

检查传播入口、人物标签、内容承诺、商业落点和长期价值观是否共同指向同一组变化。允许入口比买方更宽，不允许内容吸引的人长期无法进入任何信任或商业路径。

### 第四步：执行基础检查

使用 [positioning-and-scoring.md](positioning-and-scoring.md) 的六项基础检查：

1. 用户想起你的情况；
2. 100 题真实燃料；
3. 一句话转述；
4. 能力可信；
5. 支付邻近；
6. 伦理与生活匹配。

基础检查使用 `通过 / 暂定 / 不通过 / 未知`。不通过不能靠加权总分晋级；暂定必须绑定实验。

### 第五步：比较候选

基础检查之后，再使用 [positioning-and-scoring.md](positioning-and-scoring.md) 的九维加权评分。每个分数必须引用证据编号或标为未知。评分只用于比较，不把假设变成事实。

### 第六步：选择暂定赢家

选择一个主方向，最多保留一个受控挑战方向。明确为什么选择、最大未知、必须成立的条件、什么证据会导致修订，以及失败时先修改哪一层。

## 5. 局部修复矩阵

| 观察到的模式 | 优先判断的失败层 | 第一个动作 |
|---|---|---|
| 合格触达持续偏低 | 传播入口或内容开头 | 改具体问题、场景或形式 |
| 有播放但主页访问少 | 用户情况与相关性 | 收窄被想起的时刻 |
| 有主页访问但关注少 | 人物标签或内容承诺 | 说清持续关注会得到什么 |
| 有关注但收藏、转发弱 | 内容价值或认知增量 | 增加方法、比较、证据和可执行结果 |
| 有认同但不相信能力 | 能力证明 | 增加过程、产物、案例和结果边界 |
| 有信任评论但没有咨询 | 商业问题或下一步不可见 | 明确可解决的问题和合适行动 |
| 有咨询但不付款 | 买方、问题严重度或产品 | 测试范围、风险、价格和承诺 |
| 已付款但结果弱 | 产品与交付 | 停止放大，先修交付 |
| 反复吸引错误人群 | 用户情况或传播语言 | 修改入口和排除边界 |
| 很快写不出新内容 | 真实资产或 100 题燃料 | 回到资产发现，不继续包装定位 |

一次内容失利不触发整体重做。只有同一结构性失败在多次受控测试中重复出现，或发现伦理、身份、生活与商业方向发生实质冲突，才重新选择主定位。

## 6. 定位任务的精简交付契约

只做定位任务时，按以下顺序输出，不强制生成完整战略书：

1. 推荐定位、公开简介、用户转述句和当前信心等级；
2. 用户在什么情况下会想起你；
3. 支持推荐方向的个人资产、注意力证据、能力证明和支付证据；
4. 五层定位架构；
5. 触达人群、真正买方、不需要吸引的人和用户变化；
6. 推荐方向与最强挑战方向的基础检查和加权评分；
7. 定位失败层、保留项、最小修复和暂时不改的部分；
8. 注意力、信任和支付验证；
9. 未来 7 天与 30 天行动；
10. 什么情况说明这条路不成立。

资料不足时仍给出可完成的决定，但必须降低信心等级，并把缺口标为未知。

## 7. 停止与升级规则

- 用户只要定位诊断时，在完成上述精简交付后停止，不为显得完整而扩写整本战略书。
- 用户需要内容系统、产品阶梯、商业验证或长期路线时，升级为完整战略书。
- 定位已经清楚而问题发生在选题、内容生产、发布或交付时，停止改定位，路由到对应下游 Skill。
- 不重复维护本模块与 `positioning-and-scoring.md` 的评分数字；评分、权重和基础检查以该文件为唯一来源。
